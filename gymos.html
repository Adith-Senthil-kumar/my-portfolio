The problem

Most gym and diet apps demand sign-ups, subscriptions, and cloud sync — then stall the moment you lose signal in a basement gym.

  • Tracking is gated behind accounts and subscriptions, with your training data living on someone else’s server.
  • Apps degrade without connectivity — exactly where a gym tends to be.
  • Workout logging and nutrition usually live in separate apps, so the full picture is split in two.

What I built

GymOS keeps a complete, private training and nutrition log entirely on the device — account-free and fully usable offline.

Workout logger

Routines with training-day tabs, supersets/circuits, and per-set schemes (rep ranges, warm-ups, per-set weight/RPE). Live sessions add a rest timer, plate calculator, and automatic PR detection.

Routine builder

Drag-to-reorder, a multi-select exercise picker, duplicate-day, and a per-set target editor whose prescriptions seed the live workout.

Nutrition tracking

Calorie/macro goals (TDEE via Mifflin–St Jeor, with diet phases), a meal planner, and barcode scanning + food search via OpenFoodFacts & USDA, cached on-device.

Body & analytics

Body-measurement logging, progress photos, and an anatomical muscle-volume & recovery map, plus strength-progression and workout stats.

Offline-first data layer

A 13-table SQLite database via Drizzle ORM with on-device schema migrations at launch, plus full CSV + JSON backup & restore.

Polish

Light / Dark / AMOLED themes, kg–lb units, haptics, rest-timer notifications, and over-the-air JS updates via EAS Update.

How it works

Expo + React Native with TypeScript in strict mode. Every routine, set, meal, and photo persists to an on-device SQLite database — there is no server in the loop.

Expo + RN app Zustand stores Drizzle ORM SQLite (on-device) Migrations at launch CSV / JSON backup
ClientReact Native 0.81 / React 19 on Expo SDK 54, TypeScript strict. ~24 screens via React Navigation (bottom-tabs + stacks); react-native-paper (Material 3).
StateZustand stores for session & UI; the active-workout store persists to AsyncStorage, so an in-progress session survives an app restart.
DataSQLite (expo-sqlite) through Drizzle ORM — a 13-table schema with drizzle-kit migrations applied on first launch via useMigrations.
NutritionOpenFoodFacts + USDA FoodData Central APIs for barcode & search, with results cached on-device.
Graphicsvictory-native + Skia + SVG for charts; react-native-body-highlighter for the muscle map; Reanimated, Gesture Handler & draggable-flatlist for motion.
Build & testEAS Build (Android APK) + EAS Update for OTA JavaScript; 47 Jest (jest-expo) unit tests.

Outcome

A complete strength-and-nutrition tracker that needs no account and no backend — routines, per-set programming, macros, and progress photos all live in an on-device SQLite database and work entirely offline, with barcode nutrition pulled from OpenFoodFacts/USDA and cached locally. Designed, architected, and built solo.

Source is a private local repository and the app isn’t published — it runs as a sideloaded Android build (EAS preview). iOS is Expo-ready but not yet built.
