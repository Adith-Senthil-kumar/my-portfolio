An automated pipeline that crawls the open web and classifies cyber-threat severity in real time with a fine-tuned TinyBERT model.

The problem

Threats are planned in the open — on forums, marketplaces, and social platforms — but traditional security is reactive and reads none of it.

  • Threat intelligence is fragmented across many platforms, making it hard to gather and correlate.
  • Conventional security devices give no real-time monitoring of unstructured open-web chatter.
  • Threats evolve faster than manual analysis can keep up, given the volume and noise of web data.

What we built

A scalable CTI system that continuously watches open web sources and turns raw chatter into prioritised, actionable threat intelligence.

Automated crawling

A custom crawler periodically scans forums, marketplaces, blogs, and news portals, fetching and storing raw HTML.

NLP preprocessing

BeautifulSoup strips scripts, tags, and ads; regex normalises the text into a clean corpus for classification.

Severity classification

A fine-tuned TinyBERT model sorts threats into four levels — None, Low, Medium, High — for prioritised response.

Keyword pre-filter

A rule-based stage tags category terms (hacking, malware, fraud, weapons) to cut model load before inference.

Dashboard & analytics

A Flask dashboard visualises results with charts, filters, and severity highlighting for live monitoring.

Automated alerts

When severity crosses High, the system triggers instant mail / SMS alerts so teams can react fast.

Architecture

A multi-stage pipeline from raw web pages to actionable intelligence, deployed on cloud infrastructure for scale.

IngestionCustom Python crawler + scraper pulling surface-web content via HTTP at ~45 pages/min.
NLPBeautifulSoup cleaning, a rule-based keyword filter, and a fine-tuned TinyBERT transformer (max length 512).
StorageMongoDB holds structured records — URL, extracted text, category, severity, and timestamp.
DeliveryA Flask dashboard for analytics, plus automated mail / SMS alerts on high-severity hits.

The model

Threat text is short, noisy, and multi-class — a good fit for a small, fast transformer rather than a heavyweight LLM.

  • TinyBERT, fine-tuned for four-level severity classification (None / Low / Medium / High) at 512-token inputs.
  • A rule-based keyword pre-filter drops irrelevant pages first, cutting inference load.
  • Result: 90.2% accuracy / 0.924 AUC-ROC at 18.4 ms inference — beating Naive Bayes (80.3%), Logistic Regression (84.5%), Random Forest (85.2%), and LSTM (86.7%).
  • The crawler sustained 90.4% success over 9,560 pages while removing 80.1% of noise.

Outcome

A robust, scalable, end-to-end threat-intelligence system that processes open-web content in near real time — peer-reviewed and published at CCIC 2026.
